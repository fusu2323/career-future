### T-01-02 岗位数据导入与清洗
**时间**: Day 3-5
**负责人**: 后端

**任务内容**:
- 编写 Excel/CSV 数据读取脚本
- 实现数据清洗逻辑（去重、缺失值处理、异常值检测）
- 数据标准化（薪资范围统一、地名字段统一等）
- 将清洗后数据存入数据库

**交付物**:
- [ ] data/raw/ 目录存放原始数据
- [ ] data/processed/ 目录存放清洗后数据
- [ ] scripts/clean_data.py 清洗脚本
- [ ] 清洗后数据 JSON/SQLite 文件

**验收标准**:
```python
# 1. 清洗脚本可执行
python scripts/clean_data.py --input data/raw/xxx.xls --output data/processed/jobs.json

# 2. 清洗结果校验
- 原始数据：10000 条
- 清洗后：≥9500 条（去重后）
- 缺失值：关键字段（职位名称、职位描述、薪资）缺失率 < 1%
- 重复率：≤2%

# 3. 数据格式校验
- 薪资范围统一为 [min, max] 格式
- 城市字段标准化（如"北京"统一，不出现"北京市"/"北京地区"）
```

### T-01-05 知识图谱构建（Neo4j）
**时间**: Day 13-14
**负责人**: 后端

**任务内容**:
- 设计图谱 Schema（岗位节点、关系类型）
- 构建垂直晋升关系（初级→高级→专家）
- 构建横向换岗关系（至少 5 个岗位，每岗位≥2 条路径）
- 实现图谱查询 API

**交付物**:
- [ ] scripts/build_graph.py 图谱构建脚本
- [ ] Neo4j 图谱数据库（含节点和关系）
- [ ] /api/graph/career-path 图谱查询 API

**验收标准**:
```cypher
# 1. 节点数量
MATCH (n) RETURN count(n)  # ≥100 个岗位节点

# 2. 垂直关系验证
MATCH (j:Job {name: "初级前端工程师"})-[:PROMOTE_TO]->(s:Job)
RETURN s.name  # 应返回"高级前端工程师"等

# 3. 横向关系验证
MATCH (j:Job {name: "数据分析师"})-[r:TRANSITION_TO]->(t:Job)
RETURN t.name, r.required_skills  # ≥2 条换岗路径

# 4. API 测试
GET /api/graph/career-path?job_id=xxx&type=vertical
# 返回晋升路径节点列表

GET /api/graph/career-path?job_id=xxx&type=horizontal
# 返回换岗路径及所需补充技能
```
