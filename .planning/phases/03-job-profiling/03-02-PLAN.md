## Step 1: Load Data and Discover Job Types
- Load jobs_cleaned.json into pandas DataFrame

- Print selected job types with record counts

## Step 2: Two-Step LLM Profile Generation (for each job type)
For each job type:
a) Collect all records for that job type
b) Build details_text by truncating each job_detail_cleaned to 400 chars, prepend "城市:{city}|薪资:{salary_min}-{salary_max}|详情:{detail}"
c) EXTRACTION_PROMPT (Step 1 - structured extraction):
```
你是一名HR数据分析师。从以下{job_type}岗位的招聘数据中，提取关键统计信息。

岗位数据（共{count}条）：
{job_details}

请以JSON格式输出：
{{
  "job_type": "{job_type}",
  "total_records": {count},
  "skill_frequency": {{"技能名": 出现次数, ...}},
  "avg_salary_min": 平均月薪下限,
  "avg_salary_max": 平均月薪上限,
  "education_requirements": {{"学历": 出现次数, ...}},
  "experience_years_range": "X-Y年",
  "top_companies": ["公司名", ...],
  "common_responsibilities": ["职责1", "职责2", ...],
  "certificates_mentioned": {{"证书名": 次数, ...}}
}}
```
d) SYNTHESIS_PROMPT (Step 2 - 7-dimension synthesis):
```
你是一名职业规划专家。基于以下{job_type}岗位的招聘数据分析，生成标准7维画像。

分析数据：
{extracted_stats}

请以JSON格式输出7维画像：
{{
  "job_type": "{job_type}",
  "professional_skills": {{
    "core_skills": ["技能1", "技能2", ...],
    "soft_skills": ["技能A", "技能B", ...],
    "tools_frameworks": ["工具/框架1", ...]
  }},
  "certificate_requirements": {{
    "required": ["证书1", ...],
    "preferred": ["证书A", ...]
  }},
  "innovation_ability": 1-5,
  "learning_ability": 1-5,
  "stress_resistance": 1-5,
  "communication_ability": 1-5,
  "internship_importance": 1-5,
  "summary": "一句话概括该岗位的核心特征"
}}
```
e) Use tiktoken to count tokens. If extraction prompt > 3500 tokens, truncate details or reduce batch size.
f) Call DeepSeek API with temperature=0.1 for extraction, temperature=0.2 for synthesis
g) Parse JSON responses. On parse failure, retry once with stricter prompt.
h) Add source_record_count, avg_salary_min, avg_salary_max to final profile

## Step 3: Save JSON
- Write all profiles to data/processed/job_profiles.json with ensure_ascii=False, indent=2
- Print confirmation with profile count

## Step 4: Write to Neo4j (optional - warn on failure)
- Connect to Neo4j with GraphDatabase.driver
- For each profile, MERGE JobProfile node with all properties
- Print confirmation or warning

## Error Handling
- If DEEPSEEK_API_KEY not set, raise ValueError with clear message
- If Neo4j write fails, print warning but don't fail (JSON is primary output)
- If LLM call fails for a job type, log error and continue with remaining types
