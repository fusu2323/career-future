  <action>
    Create `app/services/__init__.py` (empty).
    Create `app/services/llm_service.py` implementing:

    **Timeout constants (per D-03):**
    ```
    TIMEOUTS = {"profile": 15.0, "match": 20.0, "report": 45.0}
    ```

    **`is_retryable_http_error` function:**
    - Returns `True` for `APITimeoutError` (per D-02)
    - Returns `True` for `APIStatusError` where `status_code >= 500 or status_code in (429, 503)` (per D-02)
    - Returns `False` for `APIStatusError` with `status_code < 500` (do NOT retry 400/401 per D-02)
    - Returns `False` for other exceptions

    **`_call_with_retry_sync` function** — tenacity-decorated synchronous wrapper:
    - Decorator: `retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=4), retry=retry_if_exception_type((APITimeoutError, APIStatusError)), reraise=True)` — gives 1s/2s/4s intervals per D-02
    - Calls `client.chat.completions.create(model="deepseek-chat", messages=messages, response_format={"type": "json_object"}, timeout=timeout, **kwargs)`
    - Catches `APIStatusError`, checks `is_retryable_http_error`, re-raises non-retryable immediately

    **`generate_structured` async function** — JSON parse retry loop per D-04:
    - Accepts `task_type, client, prompt, temperature=0.1, max_tokens=None`
    - Builds `messages = [{"role": "user", "content": prompt}]`
    - Sets `timeout = TIMEOUTS[task_type]` (raises KeyError for unknown task_type)
    - Loop `attempt in range(3)` (initial + 2 retries):
      - Inside asyncio timeout context: call `_call_with_retry_sync(client, messages, timeout, temperature=temperature, max_tokens=max_tokens)`
      - Extract `content = response.choices[0].message.content`
      - Try `json.loads(content)`, return dict on success
      - On `JSONDecodeError`: if `attempt == 2`, raise `LLMJSONParseError(f"Failed after 3 parse attempts, raw: {content[:200]}")`; else continue loop
      - On non-retryable HTTP error or `asyncio.TimeoutError`: propagate immediately

    **Note:** The tenacity decorator handles HTTP retry (3 attempts with backoff). The JSON parse retry is a separate inner loop of up to 2 additional calls. Total max calls = 3 HTTP retries x 3 JSON retries = 9 in worst case, but per D-02/D-04 design: HTTP retry exhausts first, then JSON parse retry on each retry.
  </action>
