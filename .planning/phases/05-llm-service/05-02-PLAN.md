  <action>
    Create `app/routers/__init__.py` exporting `llm_router` and `health_router`.

    Create `app/routers/llm.py`:
    - `router = APIRouter(prefix="/llm", tags=["LLM"])`
    - Import `generate_structured` from `app.services.llm_service`
    - Import `get_deepseek_client` from `app.clients.deepseek`
    - Import all `LLM*Error` exceptions from `app.exceptions.llm_exceptions`
    - Import `LLMGenerateRequest` and `LLMGenerateResponse` from `app.models.llm_models`
    - Import `OpenAI` from `openai`

    **Endpoint 1: POST `/llm/profile/generate`**
    - Path parameter: none (task_type is implicit as "profile")
    - Request body: `LLMGenerateRequest`
    - Dependency: `client: OpenAI = Depends(get_deepseek_client)`
    - Logic:
      - Call `await generate_structured("profile", client, request.prompt, request.temperature, request.max_tokens)`
      - Return `LLMGenerateResponse(success=True, data=result, task_type="profile", attempt_count=1)`
    - Error handling:
      - `LLMTimeoutError` -> `HTTPException(status_code=504, detail=str(e))`
      - `LLMJSONParseError` -> `HTTPException(status_code=422, detail=str(e))`
      - `LLMRetryExhaustedError` -> `HTTPException(status_code=502, detail=str(e))`
      - `Exception` -> `HTTPException(status_code=500, detail=f"LLM service error: {e}")`

    **Endpoint 2: POST `/llm/match/analyze`**
    - Same structure as profile endpoint but task_type="match"
    - Returns `LLMGenerateResponse(..., task_type="match", ...)`

    **Endpoint 3: POST `/llm/report/generate`**
    - Same structure as profile endpoint but task_type="report"
    - Returns `LLMGenerateResponse(..., task_type="report", ...)`

    All three endpoints share the same error handling pattern. DRY principle: consider a helper `_handle_llm_error(e)` function that maps exception types to HTTP status codes.
  </action>
  <verify>
    <automated>python -c "from app.routers.llm import router; print(f'routes: {[r.path for r in router.routes]}')"</automated>
  </verify>
  <done>Router has exactly 3 POST routes: /llm/profile/generate, /llm/match/analyze, /llm/report/generate. Each returns LLMGenerateResponse with success=True on valid input.</done>

    Create `app/routers/health.py`:
    - `router = APIRouter(prefix="", tags=["Health"])`

    **GET `/health`**
    - Returns `{"status": "ok", "service": "llm-service"}` with `status_code=200`
    - No dependencies (lightweight liveness check)

    **GET `/health/ready`**
    - Dependency: `client: OpenAI = Depends(get_deepseek_client)`
    - Performs a lightweight DeepSeek API probe: `client.chat.completions.create(model="deepseek-chat", messages=[{"role":"user","content":"hi"}], max_tokens=5, timeout=5.0)`
    - Returns `{"status": "ready", "service": "llm-service"}` on success
    - On any exception: returns `HTTPException(status_code=503, detail="DeepSeek not ready")`
    - Note: Uses max_tokens=5 and timeout=5.0 to keep the probe fast
  </action>
