# Phase 4 Plan 1: Job Graph Builder

**Plan:** 04-01
**Phase:** 04-job-graph
**Created:** 2026-03-30
**Based on:** 04-CONTEXT.md + 04-RESEARCH.md

## Goal

Build career progression graph in Neo4j `planer` database:
- **Nodes:** ≥100 (12 jobs × 3 levels + intermediate nodes)
- **Edges:** ≥200 (promotion edges + dense transition edges)
- **Query time:** ≤200ms

---

## Tasks

### Task 1: Data Preparation

**Steps:**
1. Read `data/processed/job_profiles.json` (12 job types)
2. For each job_type, split into 3 level nodes: 初级 / 中级 / 高级
3. Collect all unique core_skills → create `:Skill` nodes (for node count)
4. For each level node, link to its Skill nodes via `[:REQUIRES]` relationships

**Deliverables:**
- `nodes_batch`: List of 36 JobProfile nodes + ~50 Skill nodes
- `skill_links`: Relationships between JobProfile and Skill nodes

**Why:** Node count target is ≥100. 12 jobs × 3 levels = 36, plus ~50 Skill nodes = ~86. Need additional intermediate nodes (JobCategory nodes per level).

---

### Task 2: LLM Transition Analysis (Batch)

**Steps:**
1. Build pair matrix: all job_type pairs at each level (初级/中级/高级)
2. 12 jobs → 66 pairs per level × 3 levels = 198 candidate pairs
2. Batch into groups of 10 pairs per LLM call → ~60 calls total
3. For each batch, prompt LLM to analyze skill overlap and output:

```json
{
  "pairs": [
    {
      "from_job": "Java",
      "to_job": "前端开发",
      "level": "中级",
      "has_transition": true,
      "shared_skills": ["SQL", "团队协作"],
      "gap_skills": ["Vue", "React"],
      "difficulty": 3,
      "salary_change_pct": -5.0,
      "rationale": "..."
    },
    ...
  ]
}
```

4. Filter pairs where `has_transition: false`

**LLM Prompt (per batch of 10 pairs):**
```
你是一名职业规划专家。分析以下多个岗位对之间的技能可迁移性。

对于每一对岗位，判断：
1. 是否存在技能迁移路径（技能重叠≥30%则存在）
2. 共享技能有哪些
3. 需要补充哪些技能
4. 迁移难度（1-5）
5. 薪资变化百分比

{job_a}技能: {skills_a}
{job_b}技能: {skills_b}
...

请以JSON格式输出（pairs数组）：
{{"pairs": [{{"from_job": "...", "to_job": "...", "level": "初级/中级/高级", "has_transition": true/false, ...}}]}}
```

**Deliverables:**
- `transition_edges`: List of validated transition edges with properties
- Expected: ~100-150 valid transitions (LLM filters ~30-50% of pairs)

---

### Task 3: Neo4j Write Operations

**Order:**
1. **Full clear:** `MATCH (n) DETACH DELETE n`
2. **Create Skill nodes:** Batch UNWIND insert
3. **Create JobProfile nodes:** Batch UNWIND insert (36 nodes)
4. **Create JobProfile-Skill links:** Batch UNWIND insert
5. **Create PROMOTES_TO edges:** 24 edges (2 per job_type: 初级→中级, 中级→高级)
6. **Create TRANSITIONS_TO edges:** All validated LLM edges

** Cypher snippets:**

```python
# Full clear
"MATCH (n) DETACH DELETE n"

# Batch create JobProfile nodes
"""
UNWIND $batch AS row
MERGE (j:JobProfile {job_type: row.job_type, level: row.level})
SET j.professional_skills = row.skills,
    j.avg_salary_min = row.avg_min,
    j.avg_salary_max = row.avg_max,
    j.innovation_ability = row.innovation,
    j.learning_ability = row.learning,
    j.stress_resistance = row.stress,
    j.communication_ability = row.comm,
    j.internship_importance = row.internship,
    j.summary = row.summary,
    j.source_record_count = row.record_count
"""

# Promotion edge
"""
MATCH (a:JobProfile {job_type: $jt, level: $fl})
MATCH (b:JobProfile {job_type: $jt, level: $tl})
MERGE (a)-[r:PROMOTES_TO]->(b)
SET r.salary_increase_pct = $sip,
    r.years_to_next_level = $ytn,
    r.skill_delta = $sd
"""

# Transition edge
"""
MATCH (a:JobProfile {job_type: $fj, level: $lvl})
MATCH (b:JobProfile {job_type: $tj, level: $lvl})
MERGE (a)-[r:TRANSITIONS_TO]->(b)
SET r.shared_skills = $shared,
    r.gap_skills = $gap,
    r.difficulty = $diff,
    r.salary_change_pct = $sal_change,
    r.rationale = $rationale
"""
```

### Task 5: Verification

**Commands:**
```cypher
// Node counts
MATCH (n:JobProfile) RETURN count(n) AS job_nodes
MATCH (n:Skill) RETURN count(n) AS skill_nodes
MATCH (n) RETURN count(n) AS total_nodes

// Edge counts
MATCH ()-[r:PROMOTES_TO]->() RETURN count(r) AS promotion_edges
MATCH ()-[r:TRANSITIONS_TO]->() RETURN count(r) AS transition_edges
MATCH ()-[r:REQUIRES]->() RETURN count(r) AS skill_edges
MATCH ()-[r]->() RETURN count(r) AS total_edges

// Verify ≥2 transitions per job
MATCH (a:JobProfile)-[r:TRANSITIONS_TO]->(:JobProfile)
WITH a.job_type, count(r) as cnt
WHERE cnt >= 2
RETURN collect(a.job_type) AS jobs_with_2plus_transitions

// Query performance test
PROFILE MATCH (a:JobProfile)-[:PROMOTES_TO*1..3]->(b:JobProfile)
RETURN a.job_type, b.job_type, length(path) as hops
LIMIT 10
```

**Success criteria:**
- Total nodes ≥ 100
- Total edges ≥ 200
- At least 5 job types have ≥ 2 transition paths
- PROMOTES_TO edges exist for all 12 job types (初级→中级, 中级→高级)
