| test_no_filters | ✅ | 无筛选条件 |

**筛选规则**:
- **城市**: 支持部分匹配（"北京" 匹配 "北京 - 朝阳区"）
- **薪资**: 自动年/月转换，min_salary 检查 job.max >= user.min，max_salary 检查 job.min <= user.max
- **类别/行业**: 不区分大小写的部分匹配

**测试用例示例**:
```python
# 城市筛选
jobs = [
    {"city": "北京", "job_name": "Job1"},
    {"city": "上海", "job_name": "Job2"},
    {"city": "北京 - 朝阳区", "job_name": "Job3"}
]
filter_jobs(jobs, city="北京") → [Job1, Job3]

# 薪资筛选（年薪转月薪）
jobs = [
    {"salary": {"min": 120000, "max": 180000, "period": "year"}, "job_name": "Job1"},
    {"salary": {"min": 60000, "max": 96000, "period": "year"}, "job_name": "Job2"}
]
filter_jobs(jobs, min_salary=9000) → [Job1]  # 120000/12=10000 >= 9000

# 多条件筛选
filter_jobs(jobs, city="北京", min_salary=10000) → 只返回北京且薪资>=10000 的岗位
```

## 六、核心功能验证

### 6.1 推荐生成流程
```
1. 加载岗位数据 (9431 条)
   ↓
2. 应用筛选条件:
   - city: 城市部分匹配
   - min_salary/max_salary: 薪资范围 (支持年薪/月薪自动转换)
   - job_category: 岗位类别
   - industry: 行业
   ↓
3. 批量计算匹配分数:
   - S_base: 基础要求 (学历、城市、毕业年份)
   - S_skill: 专业技能 (必需/优先技能匹配)
   - S_soft: 职业素养 (沟通、团队协作等)
   - S_potential: 发展潜力 (学习能力、创新能力等)
   - S_total = w1×S_base + w2×S_skill + w3×S_soft + w4×S_potential
   ↓
4. 按总分数降序排序
   ↓
5. 返回 Top N 并应用分页
   ↓
6. 记录处理时间
```

### 6.2 筛选功能
| 筛选条件 | 匹配方式 | 示例 |
|---------|---------|------|
| city | 部分匹配 | "北京" → "北京 - 朝阳区" |
| min_salary | job.max >= user.min | min_salary=9000 → 返回 max>=9000 的岗位 |
| max_salary | job.min <= user.max | max_salary=15000 → 返回 min<=15000 的岗位 |
| job_category | 部分匹配 (不区分大小写) | "互联网" → "互联网技术" |
| industry | 部分匹配 (不区分大小写) | "软件" → "计算机软件，互联网" |

### 6.3 分页功能
- **page**: 页码（1 索引）
- **page_size**: 每页数量（默认 20）
- **total_pages**: 总页数 = (total + page_size - 1) // page_size
